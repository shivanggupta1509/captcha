Network Security Project: CAPTCHA-Based Access Control

This project implements a robust network security system designed to protect web content through CAPTCHA-based access control and automatic VPN detection. The system ensures that only authorized users can access protected pages, enhancing security against unauthorized access and potential threats.

Key Features

  • CAPTCHA Verification: Utilizes a client-side HTML5 Canvas-based CAPTCHA to verify users with IP addresses associated with VPNs, ensuring only legitimate users gain access.
  • Automatic VPN Detection: Leverages the ipapi.co API to identify VPN usage through IP geolocation, enforcing stricter access controls for VPN users.
  • Protected Content: Restricts access to this blog page, redirecting unauthorized attempts (without CAPTCHA verification) back to the verification page.
  • Seamless Non-VPN Access: Automatically grants access to users with non-VPN IP addresses, bypassing CAPTCHA for a smooth user experience.
  • Client-Side Implementation: Built entirely with HTML and JavaScript, ensuring easy deployment and compatibility without server-side dependencies.

Technical Details

The system uses a client-side approach to perform IP-based access control. Upon visiting the main page, the ipapi.co API checks the user's IP address to detect VPN usage. If a VPN is detected, a CAPTCHA is displayed, requiring the user to enter a randomly generated code. Successful verification sets a client-side flag, granting access to this blog page. Non-VPN users are redirected directly to the blog page without CAPTCHA. Direct access to the blog page is protected, redirecting unauthorized users to the verification page.
